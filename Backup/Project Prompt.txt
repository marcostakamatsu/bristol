We’re starting a new website redesign project for a brand called Bristol.

The final goal is to recreate their website completely inside WordPress using Elementor. However, before creating the sitemap, I want you to first understand the brand, its positioning, visual direction, tone, audience, and current website structure.

Brand guide:
*/Brand Guide/BRISTOL_brand-guide_250904_V2.pdf

Current website:
https://bristolmachinecompany.com/

Please follow this order:

1. Analyze the brand guide
Review the brand guide and summarize the key information we need for the website redesign, including:
- Brand positioning
- Target audience
- Tone of voice
- Visual style
- Colors, typography, imagery, and graphic elements
- Any important brand rules or restrictions
- What the website should communicate based on the brand

2. Audit the current website
First, review the existing audit materials in */Audits/. Then visit the current website and compare your findings against those documents.

Analyze:

- Content gaps
- Navigation problems
- Conversion opportunities
- Pages or sections that should be kept, removed, merged, or improved

Clearly separate confirmed findings from the existing audits and new findings from your own review.


3. Define the sitemap strategy
Before creating the final sitemap, explain your reasoning:
- What pages the new website should have
- Why each page is needed
- How the pages should be organized
- Which user journeys should be prioritized
- How the sitemap should support the brand and business goals

4. Create the full sitemap
After the analysis, develop a complete sitemap for the new website.

The sitemap must include:
- Every main page
- Subpages if needed
- Ordered sections for each page
- Purpose of each page
- Recommended CTA for each page
- Notes about content, UX, or Elementor implementation when relevant

Important:
Do not only list page names. Each page must include its full recommended section structure in order.

Please start by creating a clear plan for how you will approach this work. Do not generate the final sitemap yet. Once I approve the plan, we will execute it step by step.

Also, keep track of all important brand and website findings so we can reuse them throughout the project.

Following the approved Bristol sitemap, develop the website copy for each page using SEO best practices and the brand voice defined earlier, and save each page's copy separately on the folder */Copy

Source of truth:
Use the approved sitemap as the main structure. Do not create new pages or change the page order unless there is a clear reason and you explain it first.

Goal:
Write complete, section-by-section copy for every page in the sitemap, optimized for search, clarity, conversion, and brand consistency.

For each page, include:

1. SEO metadata
- Meta title, max 60 characters
- Meta description, max 155–160 characters
- Suggested URL slug
- Primary keyword
- Secondary keywords

2. Page structure
For every section, clearly define:
- Section name
- Section goal
- Heading level: H1, H2, or H3
- Final heading copy
- Body copy
- CTA text, when relevant
- Image placement recommendation
- Suggested image direction or visual subject
- Notes for Elementor layout, when useful

3. Heading hierarchy rules
- Use only one H1 per page
- Use H2s for main page sections
- Use H3s for cards, subsections, service items, FAQs, or grouped content
- Keep headings clear, keyword-conscious, and natural
- Do not keyword-stuff

4. Copywriting requirements
- Use modern, professional language aligned with Bristol’s brand voice
- Keep the tone confident, direct, experienced, and trustworthy
- Prioritize clarity over cleverness
- Make the copy feel specific to Bristol, not generic industrial copy
- Emphasize Bristol’s key differentiators where relevant:
  - fastening distribution
  - fabrication capabilities
  - Southern California presence
  - experience since 1975
  - certifications and traceability
  - real people answering the phone
  - fast quote support
  - product availability and reliability

5. Conversion requirements
- Each page must have a clear primary CTA
- Use “Request a Quote” where appropriate
- Include secondary CTAs only when they support the user journey
- Make CTA placement recommendations by section
